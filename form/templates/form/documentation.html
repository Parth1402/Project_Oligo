PyOligo Overview
PyOligo is an online tool for designing oligos for Assembly PCR and analyzing their properties using the Nearest Neightbhor Thermodynamics model. Oligos designed for Assembly PCR must be secondary-structure and primer dimer free, be of uniform length and must have uniform melting temperature across hybridization region. To satisy these requirements, PyOligo first optimizes the length of oligos and minimizes gloabal deviations in melting temperature across the overlapping regions. Thereafter, the program uses Zuker's algorithm to predict secondary structures, identifies primer dimer stability and warns the users when such structures might interfere with the PCR reaction.
Oligo Design
The main algorithm of PyOligo, Oligo Design, fragments the DNA sequence into uniform length oligos and optimizes their length to minimize the deivations in melting temperature of overlapping regions. By default, the program adjusts the length of oligos such that the melting temperature is within ±3°C of provided average melt temperature. This gives the program some flexibility when adjusting the length of oligos to reduce variation in melting temperature across the hybridization regions. The program uses the following equation to calculate melting temperature of DNA sequences, where ∆H° is enthalpy, ∆S° is entropy, R is gas constant and CT is concentration of oligo, provided that it is not self-complementary. 
                                 Tm = ∆H°∆S° + R ln CT
Primer Dimer
Primer Dimers are stable structures that can form through unwanted overlap between oligos. In Assembly PCR, each oligo must stably and exclusively overlap with its adjacent overlap. When oligos form stable non-exclusive overlaps with oligos from other parts of the duplex, the availability of oligo in the PCR mixture decreases. Therefore, PyOligo checks for all primer dimer structures among the oligos and warns the user if a stable primer dimer structure is determined. These structures are displayed under the Primer Dimer tab in the output. The output also provides a score for each structure and its melting temperature. Primer dimers are scored based on the number of two or more consecutive base pairs in structure. If the program displays the oligos, it is likely that these primer dimer might interfere with the PCR reaction. These primer dimers can be possibly avoided if the maximum length of the oligo is decreased or the melt temperature is increased.
Secondary Structure
Oligos can form stable secondary structures through self-hybridization. These stable DNA secondary structures can interfere with PCR reaction by lowering the availability of oligos. Therefore, it is important to identify the presence of these structures in oligos. To determine the presence of these structures, PyOligo uses Zuker's Minimum Free Energy algorithm. In the main output, these structures can be found under the secondary structure tab. This tab provides the secondary structure in dot-bracket notation, its free-energy and a link to its interactive visualization. By default, the output only contains the four most stable secondary structures. These structures may not neccessarily interfere with PCR reaction.
Troubleshoot
ERROR12 : This error occurs if the provided sequence is invalid. Make sure that the sequence only contains A, T, C, G and U base pairs. The sequence can be in uppercase, lowercase or a mixture of both. White spaces and tabes between base pairs are acceptable as they all will be removed.

ERROR14 : This error occurs due to problems in length of the sequence. The length of sequence must be greater than the maximum length of oligo to allow for fragmentation of sequence. The sequence must also be long enough such that at least two oligos of sizes greater than 15 nts cab be formed. This error can also be thrown by the program if the length of sequence exceeds 600 nts.

ERROR32 : The program will raise this exception if the provided melt temperature is not between 55°C and 70°C. This restriction is applied to ensure that sequence is properly fragmented and PCR can be carried out on the output oligos.

ERROR46 : This is a generalized error that is raised by the program when it is unable to design the oligos with the provided parameter. This error can be rectified primarily by changing the maximum length of oligos.

ERROR403 : This error arises if you try to request a resource over which you do not have access.

ERROR404 : A generalized error that is raised when the program cannot find the requested resource.

ERROR500 : This error indicates that unexpected internal server error occurred while processing the input.
